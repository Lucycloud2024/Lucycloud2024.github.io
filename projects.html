EpicBook — Production Deployment on Azure

Week 11–13 — Azure DevOps CI/CD Pipeline Project

Situation: Deploy a full-stack Node.js bookstore application (EpicBook) to Azure cloud with fully automated infrastructure provisioning and application deployment — using two separate CI/CD pipelines.

Action: Built two Azure DevOps pipelines — one for infrastructure (Terraform) and one for application deployment (Ansible). Terraform provisioned an Azure VM (Standard_D2s_v3), MySQL Flexible Server database (epicbook-prod-mysql), and all networking resources automatically. Ansible playbook handled Node.js dependency installation, application configuration via config.json, and Nginx reverse proxy setup routing port 80 to port 3000. Fixed multiple pipeline errors including SPN authentication issues, backend.tf variable restrictions, carriage return characters in subscription ID, and Nginx port configuration conflicts. Seeded the MySQL database with 54 books using custom SQL scripts (author_seed.sql and books_seed.sql).

Result: Application live at public Azure IP with Nginx serving traffic. MySQL database running with 54 books persisted. Complete end-to-end automation from code commit to production — infrastructure and application deployed via separate pipeline runs with zero manual server configuration.
